## ⚠️ 已知的局限性

### 1. LinkedIn 个人信息提取不准确
**现象**: 姓名、公司、职位显示为"未知"  
**原因**: LinkedIn 使用动态类名和 Shadow DOM，CSS 选择器难以定位  
**影响**: 不影响动态抓取和推送  
**解决方案**: 
- 短期：手动更新潜客信息
- 长期：优化 JavaScript 提取逻辑

### 2. 动态内容重复
**现象**: 同一条动态可能出现多次  
**原因**: 未实现去重逻辑  
**影响**: 报告中可能有重复内容  
**解决方案**: 在 `monitor.py` 中添加去重检查（检查数据库）

### 3. 飞书推送编码问题
**现象**: Python 脚本调用 `openclaw message` 时可能失败  
**原因**: Windows 编码问题导致 token 无法正确传递  
**影响**: 需要手动用 `message` 工具发送  
**解决方案**: 
- 方案 A：修复 subprocess 调用（复杂）
- 方案 B：使用飞书 Webhook（需配置）
- 方案 C：手动发送（当前方案）
