## 1. 安装与配置

### 1.1 环境准备

**系统要求：**
- Windows 10/11, macOS 10.15+, 或 Linux
- Python 3.8 或更高版本
- 2GB 可用磁盘空间

**检查 Python 版本：**
```bash
python --version
```

### 1.2 安装依赖

```bash
# 进入项目目录
cd linkedin-lead-monitor

# 安装 Python 依赖
pip install -r requirements.txt

# 安装 Chromium 浏览器
playwright install chromium
```

### 1.3 配置环境变量

**步骤：**

1. 复制配置模板
   ```bash
   cp .env.example .env
   ```

2. 编辑 `.env` 文件
   ```bash
   # Windows
   notepad .env
   
   # macOS/Linux
   nano .env
   ```

3. 填写配置
   ```bash
   # LinkedIn 账号
   LINKEDIN_EMAIL=your.email@example.com
   LINKEDIN_PASSWORD=your_password
   
   # 飞书用户 ID
   FEISHU_USER_ID=ou_xxxxxxxxxxxxxxxxxxxxxxxxx
   ```

**获取飞书用户 ID：**
1. 打开飞书
2. 点击头像 → 设置
3. 个人信息 → 用户 ID

### 1.4 验证配置

```bash
python scripts/check_config.py
```

**预期输出：**
```
✓ 依赖检查
✓ 配置文件
✓ 数据库
✓ 浏览器
✅ 所有检查通过！
```

## 2. 快速开始

### 2.1 登录 LinkedIn

```bash
python scripts/cli.py login
```

**操作：**
1. 浏览器会自动打开
2. 在浏览器中登录 LinkedIn
3. 登录成功后关闭浏览器
4. 会话会自动保存

### 2.2 添加第一个潜客

```bash
python scripts/cli.py add \
  --url "https://www.linkedin.com/in/潜客 URL" \
  --name "潜客姓名" \
  --priority high
```

**参数说明：**
- `--url`: LinkedIn 个人主页 URL
- `--name`: 潜客姓名（用于标识）
- `--priority`: 优先级（high/medium/low）

**示例：**
```bash
python scripts/cli.py add \
  --url "https://www.linkedin.com/in/john-doe-123456" \
  --name "John Doe" \
  --priority high
```

### 2.3 开始监控

**单次检查：**
```bash
python scripts/cli.py check
```

**持续监控：**
```bash
python scripts/cli.py watch --interval 30
```

**按 Ctrl+C 停止监控**

## 3. 日常使用

### 3.1 查看潜客列表

```bash
python scripts/cli.py list
```

**输出示例：**
```
================================================================================
姓名                 公司                 职位                 优先级       
================================================================================
🔴 John Doe         TechCorp           Marketing Director   high      
🟡 Jane Smith       StartupXYZ         CEO                  medium    
================================================================================
共 2 个潜客
```

### 3.2 检查特定潜客

```bash
python scripts/cli.py check --name "John Doe"
```

### 3.3 更新潜客信息

```bash
# 先删除
python scripts/cli.py remove --name "John Doe"

# 再添加（带更新的信息）
python scripts/cli.py add --url "URL" --name "John Doe" --company "New Company" --priority high
```

### 3.4 查看监控历史

```bash
# 全部历史
python scripts/cli.py history

# 特定潜客
python scripts/cli.py history --lead "John Doe"

# 查看统计
python scripts/cli.py stats
```

## 4. 高级功能

### 4.1 批量导入潜客

**创建 CSV 文件（leads.csv）：**
```csv
name,url,company,position,priority
John Doe,https://www.linkedin.com/in/john-doe,TechCorp,Marketing Director,high
Jane Smith,https://www.linkedin.com/in/jane-smith,StartupXYZ,CEO,medium
```

**导入：**
```bash
python scripts/cli.py import --file leads.csv
```

### 4.2 导出潜客数据

```bash
# CSV 格式
python scripts/cli.py export --format csv --output leads_backup.csv

# JSON 格式
python scripts/cli.py export --format json --output leads_backup.json
```

### 4.3 自定义监控间隔

```bash
# 每 15 分钟检查（高优先级潜客）
python scripts/cli.py watch --interval 15

# 每 60 分钟检查（低优先级潜客）
python scripts/cli.py watch --interval 60
```

### 4.4 后台运行（Linux/macOS）

```bash
# 使用 nohup
nohup python scripts/cli.py watch --interval 30 > monitor.log 2>&1 &

# 查看日志
tail -f monitor.log

# 停止监控
ps aux | grep "cli.py watch" | grep -v grep | awk "{print $2}" | xargs kill
```

### 4.5 后台运行（Windows）

**使用 PowerShell：**
```powershell
Start-Process python -ArgumentList "scripts/cli.py watch --interval 30" -WindowStyle Hidden
```

## 5. 故障排查

### 5.1 浏览器启动失败

**症状：**
```
Error: browserType.launch: Executable doesn't exist
```

**解决：**
```bash
# 重新安装 Chromium
playwright install chromium

# 验证安装
python -c "from playwright.sync_api import sync_playwright; print('OK')"
```

### 5.2 LinkedIn 登录失败

**症状：**
```
[ERROR] 登录失败
```

**解决：**
1. 检查 `.env` 中的账号密码
2. LinkedIn 可能要求验证码
3. 手动登录：`python scripts/cli.py login`

### 5.3 飞书推送失败

**症状：**
```
✗ 飞书消息发送失败
```

**解决：**
1. 检查 `FEISHU_USER_ID` 是否正确
2. 确保 OpenClaw 飞书集成已配置
3. 或配置 `FEISHU_WEBHOOK_URL`

### 5.4 动态抓取为空

**症状：**
```
[WARN] 未找到动态内容
```

**解决：**
1. 确保 LinkedIn 已登录
2. 潜客可能没有公开动态
3. 检查网络连接

### 5.5 数据库错误

**症状：**
```
sqlite3.OperationalError: no such table: leads
```

**解决：**
```bash
# 重新初始化数据库
python scripts/cli.py setup
```

## 6. 最佳实践

### 6.1 潜客选择策略

**高优先级潜客：**
- ✅ 目标公司的决策者（VP/Director/Manager）
- ✅ 采购部门负责人
- ✅ 近期有互动但未转化的潜客

**中优先级潜客：**
- ✅ 行业相关人士
- ✅ 潜在合作伙伴
- ✅ 竞争对手公司员工

### 6.2 监控频率建议

| 潜客类型 | 检查间隔 | 推送策略 |
|----------|----------|----------|
| 高优先级 | 15-30 分钟 | 即时推送 |
| 中优先级 | 30-60 分钟 | 即时推送 |
| 低优先级 | 2-4 小时 | 日报汇总 |

### 6.3 跟进时机

**最佳时机：**
- 🎯 职位变动后 24 小时内
- 🎯 公司动态后 48 小时内
- 🎯 内容发布后及时互动

**跟进话术：**
```
祝贺消息 + 价值主张 + 行动号召

示例：
"恭喜您晋升为 Marketing Director！
Kingsway Video 帮助 B2B 企业通过视频营销提升转化率...
有兴趣聊聊吗？"
```

### 6.4 数据备份

**定期备份数据库：**
```bash
# 每周备份
cp data/leads.db data/leads_backup_$(date +%Y%m%d).db
```

```bash
python scripts/cli.py status
```

**重新登录：**
```bash
python scripts/cli.py login
```
